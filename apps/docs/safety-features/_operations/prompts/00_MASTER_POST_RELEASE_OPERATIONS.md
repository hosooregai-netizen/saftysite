# Step 29 Master Prompt: Post-Release Operations & Monitoring

```text
너는 SaaS/ERP 프로젝트의 post-release 운영 문서를 만드는 시니어 SRE/QA lead다.

목표:
릴리즈 후 운영, 모니터링, incident response, rollback/hotfix, billing reconciliation, OAuth token operations, public share audit, support playbook, docs maintenance 문서를 작성하라.

반드시 포함할 운영 영역:
- report export and billing
- Toss confirm/webhook/ledger
- Workspace auth and guest import
- Gmail connect/token/sync
- webhard public share
- photo-album and directory data
- docs/registry maintenance

요구사항:
1. daily/weekly/monthly 운영 점검을 정의하라.
2. P0/P1 incident 기준을 정의하라.
3. rollback/hotfix runbook을 작성하라.
4. billing reconciliation 기준을 작성하라.
5. OAuth token health check 기준을 작성하라.
6. public share audit 기준을 작성하라.
7. support playbook을 작성하라.
8. docs maintenance 기준을 작성하라.
9. 앱 코드는 수정하지 마라.

완료 기준:
release 후 운영자가 점검/장애/핫픽스/문서 업데이트를 수행할 수 있다.
```
