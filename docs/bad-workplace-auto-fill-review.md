## 현재 사용할 수 있는 원본 데이터

### A. 현장/본사 마스터

`SafetySite`, `headquarter_detail`에서 아래 값을 바로 가져올 수 있다.

- 현장명
- 사업장개시번호
- 공사기간
- 공사금액
- 책임자명
- 책임자 연락처
- 현장 주소
- 회사명
- 면허번호
- 사업자등록번호
- 사업장관리번호
- 본사 주소

### B. 원본 기술지도 보고서

선택된 `InspectionSession`에서 아래 값을 가져올 수 있다.

- 지도일
- 보고서 작성자
- 공정률
- 실시횟수
- 문서 7 지적사항

또한 `document4FollowUps`에는 아래 값이 있다.

- `sourceFindingId`
- `guidanceDate`
- `confirmationDate`
- `result`

즉, 지적사항별 `이행 확인일`은 이론상 자동 연결 가능하다.

### C. 사용자/조직 정보

현재 로그인 사용자(`SafetyUser`)에서는 아래 값만 확실하다.

- 이름
- 전화번호
- 소속명(`organization_name`)

하지만 현재 현장 목록의 `assigned_user` 요약 모델에는 전화번호가 없다. 따라서 "배정된 담당 요원 연락처"를 현장 데이터만으로는 정확히 자동 채울 수 없다.

## 핵심 발견사항

### 1. `이행 확인일`은 현재 잘못 자동 채워지고 있다

- 현재 로직은 `confirmationDate = guidanceDate`로 넣는다.
- 하지만 문서 의미상 `이행 확인일`은 후속 확인 날짜다.
- 코드베이스에는 이미 `document4FollowUps.confirmationDate`와 `sourceFindingId`가 존재한다.
- 따라서 이 값은 비어 있으면 비워 두거나, 같은 `sourceFindingId`를 가진 follow-up에서 가져와야 맞다.

### 2. `담당 요원(연락처)`는 현재 사용자 기준이라 관리자 화면에서 틀릴 수 있다

- 현재 초안 생성은 `currentUser`를 보고 `reporterName`, `assigneeContact`를 만든다.
- 작업자가 작성할 때는 맞을 수 있지만, 관리자가 들어오면 관리자 이름/전화가 들어갈 수 있다.
- 문서 기준으로는 "이번 취약 현장을 지도한 담당 요원"이 들어가야 하므로, 최소한 원본 보고서 작성자 우선으로 맞추는 편이 안전하다.

### 3. `대표자`는 현재 자동값이 의미상 맞지 않다

- 문서 라벨은 `건설재해예방전문지도기관 대표자`다.
- 현재 기본값은 작성자 이름이다.
- 작성자와 기관 대표자는 다른 개념이므로, 현재 값은 임시값에 가깝다.
- 대표자 마스터가 없다면 자동 채우지 말고 수동 입력으로 두는 쪽이 더 정직하다.

### 4. `지방노동청(지청)장`은 현재 자동화 근거가 없다

- 현장 주소는 있으나 관할 노동청 매핑 테이블이 없다.
- 이 값은 자동 불러오기 대상이 아니라 "추후 관할 매핑 구현 전까지 수동 입력"으로 보는 것이 맞다.

### 5. `공정률`은 빈 값일 때 폴백 계산이 필요하다

- 화면에는 공정률 필드가 있지만, 초안 생성은 `document2Overview.progressRate`만 본다.
- 같은 파일에 이미 `formatSessionProgressRateDisplay()`가 있고, 진행률 계산 로직도 있다.
- 원본 보고서에 수기 공정률이 비어 있어도 화면 초안에는 계산값이 들어가도록 보강할 수 있다.

## 2차 결론

### 바로 자동 불러와야 하는 항목

- 현장/본사 마스터 대부분
- 원본 보고서 지도일
- 원본 보고서 지적사항의 법규/위험요인/개선지도 사항
- 첨부서류 고정 문구

### 자동 불러오되 로직을 수정해야 하는 항목

- 이행 확인일
- 담당 요원
- 담당 요원 연락처
- 공정률
- 불이행 사항

### 현재는 수동 입력으로 두는 것이 맞는 항목

- 건설재해예방전문지도기관 대표자
- 지방노동청(지청)장

## 다음 구현 권장 순서

1. `sourceFindingId` 기준으로 `document7Findings`와 `document4FollowUps`를 조인한다.
2. 항목별 `confirmationDate`와 `불이행 사항`을 follow-up 결과 우선으로 채운다.
3. `담당 요원`은 원본 보고서 작성자 또는 현장 배정 요원 기준으로 분리한다.
4. 요원 연락처는 사용자 상세 조회나 배정 사용자 확장 API가 있어야 정확 자동화 가능하다.
5. 대표자/지방관서는 마스터 데이터 또는 관할 매핑 테이블이 생길 때 자동화한다.
