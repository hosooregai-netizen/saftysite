# Assignment & Access Scope Hardening

## 목표

사용자별 사업장/현장 접근 범위와 배정 UX를 명확히 한다.

| 사용자 | 접근 범위 |
|---|---|
| 관리자 | workspace 내 모든 사업장/현장 |
| 사업장 배정 사용자 | 배정된 사업장과 하위 현장 |
| 현장 배정 사용자 | 직접 배정된 현장 |
| 미배정 사용자 | 빈 목록 또는 로그인/배정 안내 |

## Assignment UX

사업장 상세에는 배정된 지도요원, 배정 추가/해제, 하위 현장 수, 최근 보고서/사진첩 진입 링크를 표시한다. 현장 상세에는 담당자, 지도요원 배정 상태, 배정 추가/해제, 최근 기술지도 이력을 표시한다.

## 보안 기준

- workspace 밖 사용자에게 배정할 수 없다.
- 권한 없는 사용자는 assignment UI를 볼 수 없다.
- 배정 API는 workspace_id, user_id, headquarter_id/site_id를 모두 검증한다.
