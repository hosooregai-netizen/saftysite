# Feature Spec: Photo Album

## 목적

사진첩은 현장 사진을 사업장, 현장, 지도 회차, 촬영일, 업로드 사용자 기준으로 정리하고 조회할 수 있게 하는 기능이다.

## 사용자 문제

- 현장 사진이 보고서 작성, 개인 PC, 임시 저장소에 흩어져 있다.
- 특정 사업장/현장의 사진을 회차별로 빠르게 찾기 어렵다.
- 보고서에 사용된 사진과 별도 업로드 사진의 연결성이 약하다.
- 비로그인/임시 작성 상태에서 업로드한 사진을 나중에 서버로 가져오는 기준이 필요하다.

## 핵심 사용자

- 기술지도 실무자
- 보고서 작성자
- 현장/사업장 관리자
- 내부 ERP 관리자

## 핵심 기능

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 사진 목록 | 현장 사진을 grid/list로 조회 | P0 |
| 사업장/현장 필터 | headquarter/site 기준 필터 | P0 |
| 검색 | 파일명, 현장명, 사업장명, 주소 검색 | P0 |
| 업로드 | 현장/회차를 선택해 사진 업로드 | P0 |
| 회차 수정 | 선택 사진의 지도 회차 수정 | P1 |
| 다운로드 | 단일/선택 사진 다운로드 | P1 |
| 삭제 | 선택 사진 삭제 | P1 |
| 보고서 연결 | 보고서 사진 증거 후보로 연결 | P1 |
| guest cache | 비로그인 임시 사진 보관 | P1 |

## 범위

포함:

- `/photo-album` route
- PhotoAlbumPanel source readiness
- server photo album API
- guest photo album cache
- 사업장/현장 필터
- 회차 업데이트
- 사진 다운로드/삭제
- 보고서 사진 증거와의 연결 기준

제외:

- 완전한 이미지 asset storage/S3 전환
- AI 사진 분석 자체
- 웹하드 파일 공유 구현
- 보고서 guided upload 전체 재구현

## 성공 기준

- 로그인 상태에서 서버 사진첩 목록을 볼 수 있다.
- 비로그인 상태에서 guest cache 사진첩이 동작한다.
- 사업장/현장/query 필터가 동작한다.
- 선택 사진 다운로드/삭제/회차 수정이 가능하다.
- clean build에서 PhotoAlbumPanel 관련 source missing 오류가 없어야 한다.
