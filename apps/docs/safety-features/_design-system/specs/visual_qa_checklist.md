# Visual QA Checklist

## 공통

- [ ] 화면의 primary action이 명확하다.
- [ ] 검색/필터 위치가 패턴과 일치한다.
- [ ] empty/error/loading state가 있다.
- [ ] hover/selected/focus state가 있다.
- [ ] 버튼 label이 명확하다.
- [ ] 불필요한 card nesting이 없다.
- [ ] 모바일에서 overflow가 없다.
- [ ] icon button에 aria-label이 있다.

## Webhard

- [ ] Drive-like sidebar가 있다.
- [ ] 파일 목록이 카드 안에 갇혀 있지 않다.
- [ ] list/grid 전환이 있다.
- [ ] detail panel은 optional이다.
- [ ] share status badge가 명확하다.

## Mailbox

- [ ] left folder / thread list / message viewer가 구분된다.
- [ ] 새 메일 작성 CTA가 명확하다.
- [ ] 메일 없음/계정 없음/sync 실패 상태가 구분된다.
- [ ] compose panel이 페이지 이동 없이 열린다.

## Report Workspace

- [ ] 작성 단계와 현재 상태가 보인다.
- [ ] review queue가 있다.
- [ ] export 차단 사유가 보인다.
- [ ] 저장 상태가 보인다.
