# 10_WRITE_BLOCKER_RESOLUTION_REPORT

```text
너는 blocker patch 결과를 release decision에 반영하는 QA lead다.

목표:
해결된 blocker의 resolution report를 작성하고 release decision에 반영하라.

참조:
- docs/safety-features/_blocker-patches/templates/blocker_resolution_report_template.md
- docs/safety-features/_release-candidate/specs/release_decision_report_template.md
- docs/safety-features/_release-candidate/specs/remaining_patch_sprint_plan.md

요구사항:
1. Before/After를 명확히 기록하라.
2. 변경 파일을 기록하라.
3. focused QA 결과를 기록하라.
4. related regression 결과를 기록하라.
5. remaining risk를 기록하라.
6. release hold 해제 여부를 판단하라.

완료 기준:
- release decision report가 업데이트된다.
- blocker status가 Resolved 또는 Still blocking으로 명확하다.
```
