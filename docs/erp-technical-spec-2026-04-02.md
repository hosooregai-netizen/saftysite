## 1. 문서 목적

본 문서는 2026-04-01 회의에서 논의된 ERP 요구사항을 정리한 기술 명세서 초안이다.  
핵심 방향은 `기술지도 업무를 중심으로 한 운영 ERP`를 구축하는 것이며, 특히 아래 사항을 기준으로 한다.

- 기술지도 보고서, 분기 종합보고서, 불량사업장 보고, 관제, 일정, 실적, 교육자료, 계측자료를 하나의 운영 체계로 통합한다.
- ERP는 불특정 다수에게 즉시 판매하는 범용 패키지보다는 `회사별 커스터마이징이 가능한 운영 플랫폼`으로 설계한다.
- 1차 구축 범위는 내부 운영 최적화이며, 외부 판매형 상품화는 후속 단계에서 검토한다.

## 2. 제품 방향

### 2.1 핵심 제품 정의

이 시스템은 단순 보고서 작성 도구가 아니라 다음을 함께 다루는 ERP여야 한다.

- 사업장, 현장, 계약, 회차, 담당 요원, 실적의 통합 관리
- 기술지도 수행과 보고서 생성의 표준화
- 분기 단위 실적 관리와 관제 업무 지원
- 일정, 메일, 알림, 회신, 상태 추적까지 포함한 운영 관리
- 모바일 현장 수행을 지원하는 경량 웹 기능

### 2.2 핵심 판단

- `ERP를 찍어내듯 판매하는 구조`는 현재 요구사항과 맞지 않는다.
- 기관마다 문구, 항목, 프로세스, 출력 양식이 조금씩 다르므로 `기본 코어 + 기관별 설정/명칭 변경` 구조가 필요하다.
- 따라서 1차 목표는 멀티테넌트형 범용 SaaS보다 `내부용 코어 ERP + 외부사 확장 가능한 구조`에 둔다.

  - 소속 회사 데이터만 관리

## 6. 핵심 기능 명세

### 6.1 사업장/현장/계약 관리

필수 기능:

- 건설사와 그 산하 현장을 계층 구조로 관리
- 사업장 최초 등록 지원
- 운영 상태 관리: 진행중, 미착수, 중지, 재개, 완료 대기, 완료
- 계약일, 계약기간, 회차 수, 회차별 방문 기준일 관리
- 한 사업장에 여러 등록번호가 연결될 수 있는 구조 지원
- 도로명 주소 추가 및 주소 변환 지원
- 공사금액, 계약 유형(민간계약/수의계약/입찰계약), 회당 단가 저장
- Excel 업로드로 사업장/현장 생성
- Excel export 지원

K2B 관련 요구:

- K2B에서 내려받은 엑셀을 ERP에 업로드해 현장 정보를 자동 입력할 수 있어야 한다.
- K2B 원본에는 빠진 연락처/이메일 등의 항목이 있으므로 ERP 내부 보완 필드가 필요하다.
- 사업장별, 금액별, 분기별 데이터 추출이 가능해야 한다.

### 6.2 기술지도 보고서

목표:

- 현장 회차별 기술지도 보고서를 표준 양식으로 작성/출력한다.

주요 요구:

- 보고서 서식 개선: 이미지 비율, 자간, 여백, 체크박스 배치, 서명 영역, 그래프 가독성
- 현존 유해위험요인 입력 구조 정리
- 필드 정비:
  - 장소
  - 재해유형
  - 위험도(상/중/하)
  - 기인물
  - 유해위험요인
  - 개선요구사항
  - 중점관리 위험요인 및 관리대책
  - 참고법령
- `관계법령` 명칭은 `참고법령`으로 변경
- `참고자료 1, 2`는 `참고자료`로 통합
- `개선요구사항`은 2줄 기준으로 정리
- `중점관리 위험요인 및 관리대책`은 통합 필드로 운영
- 향후 진행공정에 대한 `유해/위험요인 및 안전대책` 입력 지원
- 작업공정은 텍스트 입력 + 자동 검색 보조 구조
- 공정별 위험요인/안전대책은 매칭 테이블 기반 추천 가능 구조 필요
- 기인물은 14대 기인물 체계 내에서 관리

출력 요구:

- 회차별 문서 번호/페이지 표기 필요
- 1회차 1페이지, 2회차 1페이지 식의 회차 표기 지원
- 기술지도 보고서는 PDF/HWP 등 출력 가능한 문서로 생성

### 6.3 분기 종합보고서

목표:

- 각 현장의 분기별 기술지도 실적과 위험 현황을 종합 문서로 자동 생성한다.

문서 구성:

1. 기술지도 대상사업장
2. 통계분석(누계)
3. 기술지도 이행현황
4. 향후 공정 유해위험요인 및 대책
5. 현재 공정 내 현존하는 유해위험요인 분류
6. 건설현장 12대 사망사고 기인물 핵심 안전조치

주요 요구:

- 분기 보고서는 회차 낮은 순서부터 정렬
- 기술지도 보고서를 선택한 순서대로 PDF 묶음 지원
- 발주처 수신 대상이 문서에 명시되어야 함
- 관리자 관점에서 `발송`, `수신 확인`, `회신`까지 추적해야 함
- 분기 보고서는 메일 발송 이력과 연결되어야 함
- 네이버 메일/네이버 웍스 기반 발송 및 수신 확인 검토 필요

### 6.4 불량사업장 보고

목표:

- K2B 업로드와 평가 대응을 위한 불량사업장 관련 자료를 ERP에서 관리한다.

주요 요구:

- 현장 기준으로 보고 생성
- 기존 기술지도 보고서의 지적사항을 참조해 초안 생성 가능
- K2B 업로드를 위해 필요한 자료를 다운로드 가능한 형태로 관리
- 금액 조건(예: 50억 이상) 등 분류 기준으로 목록 추출 가능
- 연말 평가 대응을 위한 보관/리스트업 기능 필요

### 6.5 관제 대시보드

목표:

- 관리자용 운영 관제 화면에서 보고서, 일정, 지연, 실적을 한 번에 확인한다.

주요 지표:

- 전체 현장 수
- 진행중 현장 수
- 미착수 현장 수
- 공사중지 현장 수
- 완료 대기 현장 수
- 보고서 작성 현황
- 분기 보고 발송/회신 현황
- 7일 내 미발송 건수
- 4일 이상 지연 건수
- 교육자료 확보 건수
- 측정자료 확보 건수
- 담당자별 실적/매출

### 6.6 일정/캘린더 관리

목표:

- 기술지도 회차와 방문 주기를 캘린더 중심으로 관리한다.

주요 요구:

- 현장 배정은 관리자가 수행
- 1개 현장에는 기본적으로 1명의 지도요원 배정
- 필요 시 중간 교체 가능
- 기술지도는 계약일 기준 14일 이내 첫 방문 규칙 반영
- 방문 주기는 계약 기간 안에서 회차별 허용 구간 기준으로 관리
- 월 2회 방문 기준의 스케줄링 지원
- 일정 변경 시 사유 입력 필수
- 일정 선택 범위 제한 필요
  - 예: 5회차 가야 하는데 2회차를 입력하지 못하게 제어
- 개인 캘린더와 전체 캘린더 모두 제공
- 관제에서 사람별 캘린더와 전체 캘린더 확인 가능

상태/사유 예시:

- 정상 진행
- 우천 취소
- 공사 중지
- 공사 재개
- 일정 연기
- 미배정

### 6.7 알림/메일/회신 관리

목표:

- 일정과 보고서 운영 누락을 줄이기 위해 메일/문자/시스템 알림을 통합한다.

필수 알림:

- 지도 실시일 기준 7일 이내 보고서 미발송 알림
- 4일 이상 지연 건 알림
- 완료 대기 1주 전 알림
- 계약 종료/공사 종료 2주 전 알림
- 공사 중지 알림
- 공사 재개 알림
- 완료증명서 발송 시점 알림

필수 기능:

- 관리자, 지도요원, 메인 대시보드에 동시 표시
- 문자와 메일을 병행할 수 있는 구조
- 빨간 불 형태의 중요 알림 표시
- 메일 발송 여부, 수신 확인 여부, 회신 여부 저장

### 6.8 모바일 웹

목표:

- 현장 수행에 꼭 필요한 기능만 모바일 웹으로 제공한다.

모바일 필수 범위:

- 현장 기본 정보 확인
- 일정 확인
- 이전 회차 지적사항 확인
- 이전 기술지도 사진 확인
- 현장에서 바로 사진 촬영 및 업로드
- 현장 담당자 서명
- 현장소장 연락처 확인

모바일 제외 또는 후순위:

- 총평 등 장문 보고서 작성
- 데스크탑 수준의 전체 문서 편집

기술 요구:

- 앱이 아닌 모바일 웹 우선
- 태블릿 호환 필수, 휴대폰도 대응
- 현장용 화면으로 재구성 필요
- 사진은 원본 저장 후 보고서 출력 시에만 축소본 사용
- 사진 메타데이터(GPS, 날짜)는 최대한 유지해야 함
- K2B 업로드용 원본 보존이 중요

### 6.9 교육자료/측정자료/콘텐츠 관리

목표:

- 평가 대응과 현장 지원을 위해 교육자료와 측정자료를 ERP 내에서 관리한다.

요구사항:

- 교육자료 업로드 및 목록 관리
- 측정자료 업로드 및 목록 관리
- 회차 종료 시점까지 자료 4건 이상 확보 여부 확인
- 관리자 대시보드에서 확보 현황 표시
- 출력물과 메일 발송 모두 지원
- 안전정보, 제재사례 등 콘텐츠 유형별 입력 폼 분리

### 6.10 실적/매출 통계

목표:

- 운영 실적과 매출을 월/분기/연 단위로 관리한다.

집계 축:

- 현장

지표:

## 8. 외부 연동 명세

### 8.1 K2B

- K2B 엑셀 다운로드 파일을 ERP에 업로드할 수 있어야 한다.
- 업로드 시 필드 매핑 규칙을 설정할 수 있어야 한다.
- ERP 내부 보완 필드와 K2B 원본 필드를 분리 관리해야 한다.
- K2B 제출용 자료를 다시 다운로드할 수 있어야 한다.

### 8.2 메일

- 네이버 메일 또는 네이버 웍스 연동 검토
- 발송 이력, 수신 확인, 회신 상태 저장
- 메일 템플릿 관리 필요

### 8.3 저장소

- 기존 ERP/사내 서버가 있으면 내부 설치 우선 검토
- 없으면 외부 서버 또는 클라우드 저장소 사용
- 데이터 용량은 약 30GB/6개월 수준을 고려
- 사진 원본 보관 정책 필요

## 9. 비기능 요구사항

- 관리자/요원 권한 분리
- 현장 단위 데이터 접근 제어
- 보고서 출력 품질 확보
- 사진 원본 보존
- 대용량 첨부 저장 가능 구조
- 메일/알림 이력 감사 가능
- 엑셀 업로드 오류 복구 가능
- 장기 운영을 고려한 백업/복구 정책 필요

## 10. 1차 구축 우선순위

### P1

- 사업장/현장/계약/회차 기본 데이터 구조
- 관리자/요원 권한 구조
- 기술지도 보고서 작성/출력 안정화
- 분기 종합보고서 생성
- 캘린더 기반 일정 관리
- 알림/메일 발송 기본 기능
- K2B 엑셀 업로드

### P2

- 모바일 현장용 웹
- 교육자료/측정자료 관리
- 실적/매출 대시보드
- 완료증명서 자동화
- 공사 중지/재개 흐름

### P3

- 회사별 관리자/멀티테넌트 구조
- 외부 판매용 설정 분리
- 보고서 자동 추천 고도화
- AI 기반 요약/추천/자동 작성

## 11. 결정 사항 요약

- ERP의 본질은 `기술지도 운영 관리 시스템`이다.
- 핵심 축은 `보고서`, `관제`, `일정`, `알림`, `모바일`, `실적`, `K2B 연계`다.
- 외부 판매형 완제품보다 `내부 운영 최적화 + 회사별 커스터마이징`이 맞다.
- 모바일은 전체 ERP가 아니라 `현장 수행 최소 기능`에 집중한다.
- AI는 후속 고도화 항목으로 둔다.

5. 모바일 현장 화면 와이어프레임 작성
