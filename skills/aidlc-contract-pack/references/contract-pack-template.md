# Contract Pack Template

Use this structure for each batch doc.

## Goal

- What large change is this batch protecting?

## Scope

- What files or folders are in scope?
- What user flows must stay intact?

## Contract Pack

- feature contracts updated
- mocked smoke paths updated
- real smoke paths updated

## Validation Commands

- exact commands to run

## Implementation Record

- actual split boundaries
- actual files added or changed
- actual command results

## Residual Debt

- what is still large
- what is blocked
- what should be next
