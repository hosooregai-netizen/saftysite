# Visual QA Baseline

## 웹하드

- [ ] 기존 ERP side rail이 웹하드 화면에 상시 노출되지 않는다.
- [ ] 좌측에 Drive-like navigation이 있다.
- [ ] 중앙 파일 목록은 카드 안 테이블이 아니라 넓은 canvas에 가깝다.
- [ ] 검색, 필터, 정렬, list/grid/detail controls가 명확하다.
- [ ] 공유 상태가 badge 또는 명확한 값으로 보인다.
- [ ] 상세 패널은 optional이며 기본 화면을 과도하게 좁히지 않는다.

## 메일함

- [ ] 메일함은 Gmail/Naver형 3-pane 구조다.
- [ ] 좌측에는 폴더와 연결 계정이 있다.
- [ ] 중앙에는 thread list가 있다.
- [ ] 오른쪽에는 message viewer가 있다.
- [ ] 계정 없음, 메일 없음, 검색 결과 없음이 각각 다른 empty state로 표시된다.
- [ ] 새 메일은 floating compose panel 또는 drawer로 열린다.

## 보고서 작성

- [ ] 단계와 입력 폼이 명확하다.
- [ ] 사진/위험요인/조치사항 흐름이 끊기지 않는다.
- [ ] AI 자동작성 결과는 검토 가능한 상태로 제공된다.
