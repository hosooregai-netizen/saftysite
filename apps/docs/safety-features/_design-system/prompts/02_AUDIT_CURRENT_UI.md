# 02_AUDIT_CURRENT_UI

```text
너는 현재 UI를 기능별로 시각 감사하는 UX 디자이너다.

목표:
웹하드, 메일함, 보고서, 사업장/현장, 사진첩, 설정, 결제 화면을 디자인 시스템 기준으로 감사하라.

검사 기준:
- docs/safety-features/_design-system/specs/visual_qa_checklist.md
- docs/safety-features/_design-system/specs/layout_principles.md
- docs/safety-features/_design-system/specs/known_issues.md

요구사항:
1. 각 route별 layout pattern이 맞는지 확인하라.
2. card nesting 문제를 찾으라.
3. search/filter/action 위치가 일관적인지 확인하라.
4. empty/error/loading state를 확인하라.
5. icon button accessibility를 확인하라.
6. desktop/tablet/mobile 위험을 정리하라.
7. visual regression checklist를 기능별로 업데이트하라.

완료 기준:
- 기능별 visual issue table
- 우선순위 P0/P1/P2 분류
- 수정 프롬프트 초안
```
