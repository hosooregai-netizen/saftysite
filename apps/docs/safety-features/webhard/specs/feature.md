# Feature Spec: Webhard

## 1. Purpose

웹하드는 사용자가 업무 자료, 문서, 파일, 링크, 메모를 폴더 기반으로 정리하고, 필요한 대상에게 권한 또는 공유 링크로 전달할 수 있게 하는 파일 관리 기능이다.

## 2. User Problems

- 보고서, 현장 사진, 업무 파일이 여러 위치에 흩어져 있다.
- 사용자가 특정 폴더만 공유하고 싶어도 권한 경계가 명확하지 않으면 보안 위험이 생긴다.
- 단순 업로드 목록 UI로는 파일이 많아졌을 때 검색, 정렬, 공유, 휴지통 관리가 어렵다.
- 공개 링크를 제공할 때 공유 루트 밖 항목이 노출되면 안 된다.

## 3. Target Users

- 기술지도 보고서를 작성하는 실무자
- 사업장/현장 담당자
- 내부 관리자
- 공유 링크를 통해 자료를 확인하는 외부 수신자

## 4. Scope

### Included

- 내 드라이브 / 내 자료함
- 폴더 생성
- 파일 업로드
- 메모 생성
- 링크 생성
- 검색
- 정렬
- 리스트/그리드 보기 전환
- 항목 선택
- 상세 패널
- 이름 변경
- 이동
- 중요 표시
- 휴지통 이동/복원/영구 삭제
- 다운로드/미리보기
- 사용자/그룹/워크스페이스 권한
- 링크 공유
- 공개 공유 페이지
- 공유 폴더 하위 탐색

### Excluded for MVP

- 대용량 object storage 직접 업로드
- 파일 버전 관리
- 공동 편집
- 실시간 presence
- 댓글/멘션
- 외부 도메인 정책의 세밀한 관리자 제어

## 5. Product Principles

1. 웹하드는 일반 ERP 카드 화면이 아니라 Drive-like fullscreen file manager이다.
2. 공유 기능은 UI보다 권한 경계가 우선이다.
3. 공개 token은 공유 root와 descendants만 볼 수 있다.
4. 휴지통/삭제/폐기/만료 항목은 공유 결과에 포함하지 않는다.
5. 사용자에게 보이는 공유 상태와 실제 권한 검사 결과가 어긋나면 안 된다.

## 6. Success Criteria

- `/webhard` 진입 시 Drive-like file manager 구조가 표시된다.
- 사용자는 좌측 navigation에서 내 드라이브, 공유 문서함, 최근, 중요, 휴지통을 전환할 수 있다.
- 사용자는 폴더, 메모, 링크, 파일을 만들 수 있다.
- 사용자는 파일/폴더를 선택하고 공유, 다운로드, 이름 변경, 이동, 휴지통 작업을 할 수 있다.
- 사용자는 폴더를 `anyone_with_link` viewer로 공유할 수 있다.
- `/share/[token]`에서는 공유 root와 하위 항목만 탐색 가능하다.
- 공유 root 밖 parent_id 또는 item_id 요청은 404/403으로 차단된다.
- revoked/expired/deleted/trashed share는 데이터를 반환하지 않는다.
