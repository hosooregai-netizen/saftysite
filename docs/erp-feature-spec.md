# ERP 기능 명세 초안

## 목적

- 지도요원이 배정된 현장에서 `기술지도 보고서`, `분기 종합보고서`, `불량사업장 신고서`를 작성할 수 있어야 한다.
- 관제/관리자는 현장 단위와 요원 단위로 작성 실적과 미작성 상태를 한 화면에서 확인할 수 있어야 한다.
- 기술지도 보고서 7번(`현존 유해·위험요인 세부 지적`)은 재해사례 DB를 기반으로 개선대책, 유형, 법령 추천 품질을 높여야 한다.

## 회의 질문 기준 정리

### Q1. 재해사례 엑셀의 예방대책은 어디에 쓰나?

- 맞다. 엑셀의 `예방대책`은 기술지도 보고서 7번의 `개선 대책` 추천 DB로 쓰는 것이 가장 자연스럽다.
- 다만 13번(`재해 사례`)은 현재 단순 제목/이미지 피드 구조라, 이 DB를 그대로 다 보여주기보다는 `현장 지적사항과 가장 가까운 사례 4건 추천` 구조로 바꾸는 것이 적합하다.

### Q2. 기인물/유형이 지금보다 더 많아 보이는데?

- 현재 시스템의 14개 기인물은 유지하되, 재해사례 DB의 세부 항목을 그 14개 상위 기인물에 매핑하는 구조가 적절하다.
- 사고유형은 현재보다 확장하는 편이 맞다.
  - 권장 추가값: `떨어짐`, `맞음`, `부딪힘`, `깔림`, `매몰`, `화상`, `찔림`
  - 기존 값과 병행 사용: `추락`, `낙하`, `충돌`, `끼임`, `감전`, `전도`, `붕괴`, `화재·폭발`, `기타`

### Q3. 기인물별 법령 정보는?

- 현재는 정식 법령 DB가 아직 없으므로, 1차 구현은 `기인물 기반 대표 법령 추천`으로 처리한다.
- 이후 고객이 정리한 법령표가 오면 `기인물 + 사고유형 + 키워드` 기반 정밀 매핑으로 교체한다.

### Q4. 불량사업장 신고는 KPI인가, 대외 제출인가?

- UI/데이터 모델은 둘 다 수용 가능하게 설계해야 한다.
- 초기 기본값은 `내부 KPI 관리`로 두고, 문서 출력 포맷은 실제 통보서 양식과 동일하게 맞춘다.
- 최종 운영 정책에서 `월 1건 필수 제출`인지 `내부 관리용`인지만 관리자 설정으로 분기 가능하게 두는 편이 안전하다.

### Q5. 이전 보고서에서 항목을 가져오게 할까?

- 맞다. 불량사업장 신고서는 기존 기술지도 보고서 7번/4번 데이터에서 가져오는 방식이 가장 효율적이다.
- 추천 방식:
  - 현장 선택
  - 해당 현장의 이전 기술지도 보고서 목록
  - 보고서 내 지적사항(위험장소/위험요인/개선대책/이행상태) 선택
  - 선택값을 신고서 초안에 자동 반영

### Q6. 관리자에서는 어떻게 봐야 하나?

- 맞다. 1차 구현은 테이블 중심 관리가 가장 적합하다.
- 분기 보고서는 `현장별 작성 여부`
- 불량사업장 신고는 `요원별 월간 작성 여부`
- 두 영역 모두 `미작성`, `작성중`, `완료`, `기한경과` 상태 표시가 필요하다.

## 역할별 기능

### 지도요원

- 배정 현장에서 `기술지도 보고서` 작성
- 배정 현장에서 `분기 종합보고서` 작성
- 배정 현장에서 `불량사업장 신고서` 작성
- 기존 보고서 지적사항에서 신고서 초안 불러오기
- 문서 미리보기 및 출력

### 관리자/관제

- 현장별 기술지도/분기보고서 작성 현황 조회
- 요원별 월간 불량사업장 신고 실적 조회
- 미작성 대상 필터링
- 제출/출력 이력 조회

## 신규 기능 1: 분기 종합보고서

### 작성 대상 규칙

- 공사기간이 3개월 이상인 현장만 대상
- 분기 기준:
  - 1분기: 1~3월
  - 2분기: 4~6월
  - 3분기: 7~9월
  - 4분기: 10~12월
- 현장 공사기간과 분기가 겹치면 해당 분기 작성 대상 생성

### 화면 구성

- 지도요원 홈/현장 상세에서 `분기 보고서` 탭 추가
- 분기별 카드:
  - `2026년 1/4분기`
  - 상태: 미작성 / 작성중 / 완료
  - 마지막 저장일
  - 작성 버튼

### 문서 구조

첨부 PDF 기준 6개 섹션으로 구성한다.

1. 기술지도 대상사업장
- 기술지도 보고서 1번의 현장/본사 정보 재사용

2. 통계분석(누계)
- 해당 분기 종료일까지 누적된 기술지도 보고서 7번 데이터를 집계
- 재해형태별 분석
- 기인물별 분석

3. 기술지도 총평
- 해당 분기 기술지도 보고서 5번 총평과 7번 지적사항을 기반으로 자동 요약

4. 기술지도 이행현황
- 해당 현장의 분기 내 기술지도 보고서 목록 표
- 차수, 담당자, 실시일, 공정율, 지적 건수, 개선 건수, 비고

5. 향후 공정 유해·위험요인 및 대책
- 분기 말 기준 최신 기술지도 보고서 8번 데이터 우선 사용
- 없으면 분기 내 보고서에서 자주 나온 공정 기준으로 자동 제안

6. 주요 유해·위험작업 안전대책
- 분기 내 최다 기인물/사고유형 기반으로 대표 안전대책 1~3개 생성

### 관리자 확인 항목

- 현장명
- 공사기간
- 작성 대상 분기 수
- 작성 완료 분기 수
- 미작성 분기 수
- 마지막 작성자
- 마지막 저장일

## 신규 기능 2: 불량사업장 신고서

### 작성 대상 규칙

- 요원 1명당 월 1건 이상 작성 여부 관리
- 현장 기준이 아니라 `요원-월` 기준 KPI를 생성
- 하나의 신고서는 특정 현장과 연결

### 신고서 작성 흐름

1. 요원이 `불량사업장 신고` 메뉴 진입
2. 당월 신고 현황 확인
3. 현장 선택
4. 기존 기술지도 보고서에서 지적사항 불러오기 또는 수동 작성
5. 통보서 양식에 맞게 출력

### 양식 항목

첨부 PDF 기준으로 다음 데이터가 필요하다.

상단 기본정보:
- 기술지도 대상사업장
- 현장명
- 현장 공사사 납품번호 및 기타공사명
- 사업장개시번호
- 공사기간
- 공정률
- 공사금액
- 책임자(수신자)
- 연락처
- 주소
- 회사명
- 건설업번호
- 법인등록번호
- 사업장관리번호
- 본사주소

지도기관 정보:
- 기관명
- 등록상호명칭
- 대표자
- 점검구
- 소재지
- 기술지도 실시횟수
- 계약기간
- 담당 요원(연락처)
- 최우수

불이행 사항 표:
- 관련 법규
- 유해·위험요인
- 개선지도 사항(지도원)
- 불이행 사항(확인일)

### 추천 데이터 소스

- 기술지도 보고서 7번:
  - 위험장소
  - 유해위험요인
  - 개선대책
  - 관계법령
- 기술지도 보고서 4번:
  - 이전 지적사항 이행 여부
- 기술지도 보고서 2번:
  - 공정율
  - 실시일

### 관리자 확인 항목

- 요원명
- 연월
- 신고 건수
- 목표 충족 여부(1건 이상)
- 연결 현장명
- 마지막 제출일
- 출력 여부
