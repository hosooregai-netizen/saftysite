# Webhard Specs

이 폴더는 웹하드 기능의 명세 문서만 보관한다. 구현 프롬프트는 `../prompts/`에 둔다.

## Specs Index

| File | Purpose |
|---|---|
| `feature.md` | 기능 목적, 범위, 성공 기준 |
| `user_flows.md` | 주요 사용자 흐름 |
| `data_flow.md` | 화면, 상태, API, backend 흐름 |
| `schema.md` | 타입과 데이터 모델 |
| `api_contract.md` | API request/response 계약 |
| `permissions.md` | 권한, 링크 공유, 폴더 상속 |
| `public_share.md` | 공개 공유 페이지와 token boundary |
| `ui_ux.md` | Drive-like UI/UX 기준 |
| `validation.md` | 검증 기준 |
| `reverse_map.md` | 기능 → 코드 → API → prompt 역추적 |
| `test_scenarios.md` | 테스트 시나리오 |
| `code_inventory.md` | 현재 코드 파일 재고 |
| `known_issues.md` | 남은 이슈와 리스크 |

## Rule

이 폴더는 “무엇을 만들어야 하는가”를 설명한다. “어떻게 AI 에이전트에게 구현시킬 것인가”는 `../prompts/`에 작성한다.
